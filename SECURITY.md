# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | ✅ Yes    |

Older versions receive no security updates. Always use the latest release.

---

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

To report a security issue privately:

1. Go to the **Security** tab of this repository on GitHub.
2. Click **"Report a vulnerability"** to open a private advisory draft.
3. Fill in the details: description, steps to reproduce, potential impact, and (if available) a suggested fix.

Alternatively, email the maintainers directly (see the repository's contact info on the GitHub profile).

### What to expect

We follow a best-effort approach to vulnerability handling:

- **Acknowledgement**: We will confirm receipt as soon as reasonably possible.
- **Assessment**: We will evaluate the validity and severity of the report.
- **Resolution**: We aim to address issues in a timely manner, prioritising based on severity.
- **Credit**: With your permission, we will credit you in the release notes.

We follow [responsible disclosure](https://en.wikipedia.org/wiki/Responsible_disclosure) and ask that you do the same — please do not disclose publicly until we have had a reasonable opportunity to release a fix.

---

## Dependency Security

We run automated dependency audits via Dependabot and GitHub Actions. If you discover a vulnerability in a dependency we use, please report it both to us (so we can update) and to the upstream project.
